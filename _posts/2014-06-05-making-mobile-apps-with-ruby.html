Making Mobile Apps with Ruby

Yes, you can put it on Rails or make it light and smooth with Sinatra It used to be that if you wanted to make a smartphone app you needed to code in Objective C for iPhones and Java for Android phones. Today, there are new options for Rubyists to create apps.

Android - JRuby + Ruboto

JRuby allows Ruby to be compiled using the Java compiler. Ruby code can actually run much faster this way as JRuby can take advantage of Java benefits like multi-threading. JRuby works anywhere Java does.

Download JRuby Here.

Ruboto is a framework specializing in creating Android apps. You don't actually need it, but the Ruboto team has already done a lot of work linking in to commonly used Android resources. You can build parts of your app via the command line like in Rails.

iOS - MacRuby + RubyMotion

MacRuby is Apple's implementation of Ruby. All Mac computers ship with MacRuby installed.

RubyMotion brings Ruby code to iOS devices. Thanks to RubyMotion, ruby apps compile natively and run about as fast as if they were written in Objective C
